# 如何评价Godot
Godot4.x的表现不及Unity5的样子，但是做手游完全够用。
    Unity5作为最适合开发手游的版本也是有很多功能过剩的。
    地形、物理等大轮子应该根据项目需求轻量化，别人提供的不一定最佳。
在国内Godot没有公开的讨论热度，也没有培训班，所以大家自己组织讨论。
底层C++，脚本支持C#，可作为Unity的替代品。

# 安装Godot
在Godot的官网选择最新版本下载，解压zip后直接运行exe文件。
    我选择支持C#脚本的Godot Engine-.Net版本。

# 安装示例demo
作为新手主要是想观摩一下C#文件的编辑与使用，所以多下载几个demo看看能不能运行。
    我选择资产库项目中的Snake C#，并使用Git观察文件变化。
    网上有大量案例demo可以参考，比如：https://github.com/gdquest-demos

# 节点树
和Unity类似的节点树，有显式的场景根节点，每个节点只能挂一个脚本。
    节点图标可以快速预览节点功能。
    只要我们把节点和Unity的mono对应起来，很快就能进入工作状态了。
